[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
I start in the frontend, where I send a question to POST /run-multi-agent.
Then FastAPI hands the request to MultiAgentOrchestrator, which is the central coordinator.
First, validation_agent acts as the security checkpoint and decides whether the prompt is on-topic for Kate’s professional profile.
If it’s off-topic, the flow returns a controlled refusal message and stops expensive downstream work.
If it’s on-topic, routing_agent selects which specialists should run for that question.
resume_agent handles background, career, education, certifications, and publications.
skills_agent handles technical competencies like AI, NLP, RAG, and frameworks.
project_agent handles project-specific details like Lamabot, Babyyoda, and Ticket Classification.
Those specialist agents call MCP-style retrieval tools, specifically retrieve_resume_context, retrieve_skills_context, and retrieve_project_context, which are backed by retrieve_context and cache logic.
After specialists finish, answer_agent synthesizes a final coherent response for the user.
Then the system persists traces: question intent, latency, token estimate, and per-agent execution metadata.
Finally, the Human ✋ loop happens through thumbs up and thumbs down feedback in the UI, which we store and surface in the admin dashboard.
